Documentation de PluginsAdministration

Introduction

Ce plugin permet aux administrateurs de gérer les plugins avec une interface web. Les fonctionnalités sont:
Gérer les Plugins Installés
Rendre disponible/indisponible des plugins, consulter leurs propriétés, les désinstaller.
Installer un Plugin
Finir l'installation d'un plugin.
Gérer les Priorités
Les priorités vous permettre de reveiller les plugins dans un ordre différent de celui par défaut.

Gérer les Plugins Installés

Capture d'écran de l'interface 'Plugins installés'
1. Plugin
Description du plugin.
2. Disponible?
Rend le plugin (in)disponible.
3. Actions
Actions supplémentaires : affiche les propriétés, désinstalle le plugin.

Cette interface affiche les informations à propos des plugins installés. Ces plugins peuvent être disponibles ou indisponibles et vous avez d'autres actions à votre disposition : les désinstaller ou afficher leurs propriétés.

Il y a au moins un plugin : PluginsAdministration lui-même, qui ne peut être désinstallé ou indisponible.

Merci de noter que la désinstallation ou l'indisponibilité d'un plugin seulement via cette interface n'empèche pas l'accès à son espace web !

Les propriétés d'un plugin sont :

  • Nom,
  • Version,
  • Description,
  • Hooks écoutés,
  • Eventuellement d'autres propriétés dépendantes du plugin.

Installer un Plugin

Capture d'écran de l'interface 'Plugins pas encore installés'

Cette interface liste tous les plugins qui ne sont pas encore installés dans Codendi. Leur répertoires sont présents, mais Codendi ne sait rien à propos d'eux. Sélectionnez le plugin que vous voulez installer et il sera enregistré, si il contient un script install.sql dans le répertoire db/, celui-ci sera executé.

Gérer les Priorités

Capture d'écran de l'interface 'Priorités entre plugins'
1. Hook
Sélectionne le hook.
2. Priorités
Configure les priorités entre plugins pour le hook sélectionné.

Cette interface vous permet d'ajuster les priorités entre plugins pour un hook. La liste déroulante contient tous les hooks écoutés, les hooks intéressants (ceux qui sont écoutés par plusieurs plugins) sont signalés par une marque *.

Les priorités sont matérialisées par des entiers. Les plugins qui possèdent une priorité plus grande que les autres seront réveillés en premiers. Dans l'image ci-dessus, c'est le plugin 'Template' qui viendra en premier.
