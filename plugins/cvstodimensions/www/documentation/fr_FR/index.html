Documentation de de la passerelle CVS vers Dimensions

Introduction

Si vous gérez actuellement vos sources avec CVS, il demeure important d'archiver régulièrement les versions stables de votre projet dans 26C. A cet effet, vous pouvez utiliser cette passerelle afin de transférer les éléments gérés dans CVS vers Dimensions/26C. Cette passerelle se décompose en trois étapes :
Enregistrer les paramètres
Il faut tout d'abord enregistrer les paramètres necessaires au bon fonctionnement du transfert vers Dimensions. Ceci est à faire lors du premier transfert et après la création d'un nouveau module CVS.
Transférer un tag
Une fois les paramètres enregistrés, on peut choisir le tag à transférer, renseigner le mot de passe et lancer le transfert.
Visualiser l'historique des transferts
Une fois le tag transféré, il apparrait dans le tableau des transferts. Vous pouvez ainsi consulter tous vos anciens transferts, qu'ils aient échoués ou non.

Enregistrer les paramètres

Capture d'écran de l'interface 'Paramètres'

Avant d'effectuer un transfert, vous devez avoir préalablement enregistré les paramètres décrit ci dessous :

  • Product : il est prérempli avec le nom court de votre projet. Si le nom court ne correspond pas au product Dimensions (cas non conforme au modèle), ce paramètre peut être modifié.
  • Database : ce champs libre vous permet de renseigner la database contenant le product Dimensions.
  • Correspondance Module - Design Parts : l'ensemble des modules CVS du projet est listé. A chacun de ces modules correspond un champ libre vous permettant de renseigner le design part correspondant dans Dimensions.

Vous devez ensuite enregistrer ces paramètres en cliquant sur le bouton "Enregistrer" Ainsi lors de chaque transfert concernant votre projet, vous n'aurez pas à renseigner de nouveau ces paramètres. Si vous avez nouveaux modules dans votre projet, une nouvelle ligne correspondant à chacun des nouveaux modules apparetra avec un champs libre que vous devrez obligatoirement remplir avant d'effectuer un transfert.

Transférer un tag

Capture d'écran de l'interface 'Transfert de tags'

Après avoir enregistrer les paramètres, vous allez pouvoir effectuer votre transfert vers Dimensions. Pour cela il vous faut encore renseigner deux informations:

  • Tag : vous permet de choisir entre les différents tags transférables existants celui qui servira au transfert.
  • Mot de passe : un champ libre vous permet de saisir votre mot de passe 26C correspondant à votre login 26C identique à votre login CodeX.

Vous pouvez maintenant lancer le transfert en cliquant sur le bouton "Transférer"

Coup de pouce : tags transférables

Un tag est transférable s'il vérifie certains critères :

  • GoRoCo : pour qu'un tag soit transférable il doit être dde la forme 'GxxRyyCzz'. Les tags dont le nom contient des caractères avant ou après la chaîne "GxxRyyCzz" ne sont pas pris en compte.Les xx, yy et zz peuvent être sur un ou deux caratères numériques.
  • Tag antérieur : à partir d'un GxxRyyCzz donné, on peut importer seulement les Czz supérieurs. A partir du moment ou un workset GxxRyy est créé, on interdit la création d'un workset antérieur (Gxx inférieur ou Ryy inférieur sur un Gxx égal).
    Exemple :
    • G02R03 créé lors de l'import du tag G02R03C01
    • Si il n'existe pas de workset G02R02, on ne remonte aucun tags de type G02R02CX
    • par contre, si le workset G02R02 existe déjà (c'est-à-dire qu'on a déjà remonté un tag G02R02CX avant d'avoir remonter un tag G02R03CX), on peut importer des tags G02R02CX+n => c'est le cas d'un correction pour la maintenance.
  • Tag sur tout le projet : un tag doit être appliqué à tout le projet. Si ce n'est pas le cas, le tag apparaitra tout de même dans la liste des tags transférable. En revanche lors du transfert de CVS vers Dimensions, un dossier vide sera créé et donc transféré pour chaque module n'appartenant pas au tag.

Visualiser l'historique des transferts

Capture d'écran de l'interface 'Historique des transferts'

Une fois le transfert lancé, vous pouvez le retrouver dans l'écran "Historique". Cet écran contient la liste de tous les transferts effectués avec leur état :

  • En cours : le transfert a bien été lancé, mais il n'est pas terminé. Le tag sera provisoirement retiré de la liste des tags transférables. Il sera remi si le transfert échoue.
  • Succés : le t,
  • Transfert annulé : une tantative de transfert a été faite, mais elle a échouée. Vous trouverez à côté de cet état les raisons pour lesquelles le transfert a éhoué. Un tag dont le transfert a échoué ne sera pas supprimé de la liste des tags transférables.
