Documentation de de la passerelle SVN vers Dimensions

Introduction

Si vous gérez actuellement vos sources avec SVN, il demeure important d'archiver régulièrement les versions stables de votre projet dans 26C. A cet effet, vous pouvez utiliser cette passerelle afin de transférer les éléments gérés dans SVN vers Dimensions/26C. Cette passerelle se décompose en deux étapes :
Enregistrer les paramètres
Il faut tout d'abord enregistrer les paramètres necessaires au bon fonctionnement du transfert vers Dimensions. Ceci est à faire lors du premier transfert et après la création d'un nouveau projet SVN .
Transférer un tag
Une fois les paramètres enregistrés, on peut choisir le tag à transférer, renseigner le mot de passe et lancer le transfert.
Visualiser l'historique des transferts
Une fois le tag transféré, il apparrait dans le tableau des transferts. Vous pouvez ainsi consulter tous vos anciens transferts, qu'ils aient échoués ou non.

Enregistrer les paramètres

Capture d'écran de l'interface 'Paramètres' dans l'onglet Transfert

Avant d'effectuer un transfert, vous devez avoir préalablement enregistré les paramètres décrit ci dessous :

  • Product : il est prérempli avec le nom court de votre projet. Si le nom court ne correspond pas au product Dimensions (cas non conforme au modèle), ce paramètre peut être modifié.
  • Database : ce champs libre vous permet de renseigner la database contenant le product Dimensions.

Vous devez ensuite enregistrer ces paramètres en cliquant sur le bouton "Rafraîchir les PL" Ainsi lors de chaque transfert concernant votre projet, vous n'aurez pas à renseigner de nouveau ces paramètres. La liste des PLs est automatiquement mise à jour.

Transférer un tag

Capture d'écran de l'interface 'Transfert de tags' dans l'onglet Transfert

Après avoir enregistrer les paramètres, vous allez pouvoir effectuer votre transfert vers Dimensions. Pour cela il vous faut encore renseigner plusieurs informations:

  • Type de transfert : vous permet de choisir le type de tag à transférer, c'est à dire un tag sur un PL donné ou un tag de niveau applicatif (sur tout le projet)
  • PL : vous permet de choisir le design part pour lequel vous voulez faire le transfert(si vous avez choisi transfert par PL)
  • Tag : vous permet de choisir entre les différents tags transférables existants celui qui servira au transfert.
  • Mot de passe : un champ libre vous permet de saisir votre mot de passe 26C correspondant à votre login 26C identique à votre login Codendi.

Vous pouvez maintenant lancer le transfert en cliquant sur le bouton "Transférer"

Coup de pouce : tags transférables

Un tag est transférable s'il vérifie certains critères :

  • GoRoCo : pour qu'un tag soit transférable il doit être de la forme 'GxxRyyCzz'. Les xx, yy et zz peuvent être sur un ou deux caratères numériques.

Visualiser l'historique des transferts

Capture d'écran de l'interface 'Historique des transferts'

Une fois le transfert lancé, vous pouvez le retrouver dans l'écran "Historique". Cet écran contient la liste de tous les transferts effectués, le PL sur lesquels ils ont été effectués ainsi que leur état :

  • En cours : le transfert a bien été lancé, mais il n'est pas terminé. Le tag sera provisoirement retiré de la liste des tags transférables. Il sera remi si le transfert échoue.
  • Succés : le t,
  • Transfert annulé : une tantative de transfert a été faite, mais elle a échouée. Vous trouverez à côté de cet état les raisons pour lesquelles le transfert a éhoué. Un tag dont le transfert a échoué ne sera pas supprimé de la liste des tags transférables.
