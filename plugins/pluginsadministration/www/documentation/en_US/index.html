PluginsAdministration Documentation

Introduction

This plugin allow administrators to manage plugins with a web-based interface. The features are:
Manage Installed Plugins
Enable/disable plugins, consult their properties, uninstall them.
Install a Plugin
Finish the installation of the plugin.
Manage Priorities
If you want your plugins to be waked up in a special order, priorities are for you.

Manage Installed Plugins

Screenshot of panel 'Installed Plugins'
1. Plugin
Description of the plugin.
2. Enabled?
Enable or disable the plugin.
3. Actions
Special actions: show its properties, uninstall the plugin.

This panel displays informations about installed plugins. Those plugins can be enabled or disabled and you can do others actions : uninstall them and show properties.

There is at least one plugin: PluginsAdministration itself, which cannot be uninstalled or disabled.

Please note that uninstall or make a plugin disabled through this interface does not prevent access to its web pages!

The properties for a plugin are:

  • FullName,
  • Version,
  • Description,
  • Listened Hooks.

Install a Plugin

Screenshot of panel 'Not Yet Installed Plugins'

This panel lists all plugins that is not yet installed into CodeX. Their directories are here, but CodeX does not know about them. Select the plugin you want to install and it will be registered, if it contains a script install.sql in db/ directory, it will be executed.

Manage Priorities

Screenshot of panel 'Priorities Between Plugins'
1. Hook
Select the hook.
2. Priorities
Configure the priorities between plugins for the selected hook.

This panel allows you to ajust priorities between plugins for a hook. The select box contains all listened hook, interesting hooks (those listened to by several plugins) are distinguished by a mark *.

Priorities are realized by integers. The plugins which have a priority bigger than the others will be waked up before them. In the figure above, this is the plugin 'Template' which will come first.
